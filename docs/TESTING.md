# Методика тестирования

## Методы тестирования

### Черный ящик (Black Box Testing)

Тестирование функциональности без знания внутренней реализации системы.

#### Тестируемые сценарии:

1. **Создание студента**
   - Валидные данные
   - Данные без отчества и email
   - Невалидный пол
   - Получение студента по ID
   - Обновление данных
   - Удаление студента без заселений

2. **Заселение**
   - Валидное заселение
   - Заселение в заполненную комнату
   - Заселение с несоответствием пола

### Белый ящик (White Box Testing)

Тестирование с знанием внутренней реализации системы.

#### Тестируемые компоненты:

1. **Валидаторы**
   - Валидация имен (пустые, короткие, недопустимые символы)
   - Валидация телефонов (формат, длина)
   - Валидация email (формат, опциональность)
   - Валидация пола

2. **Внутренняя логика**
   - Получение текущей заселенности комнаты
   - Определение пола в комнате
   - Логика после выселения

## Запуск тестов

```bash
# Все тесты
python -m unittest discover tests

# Только черный ящик
python -m unittest tests.test_black_box

# Только белый ящик
python -m unittest tests.test_white_box

# С подробным выводом
python -m unittest discover tests -v
```

## Покрытие тестами

- Модели данных: ~80%
- Валидаторы: ~90%
- Бизнес-логика: ~75%
