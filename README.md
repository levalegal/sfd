# Информационная система управления студенческим общежитием

Система управления для автоматизации процессов заселения и выселения студентов в общежитии.

## Описание

Программное обеспечение предназначено для управления:
- Студентами
- Комендантами
- Корпусами общежития
- Комнатами
- Заселениями и выселениями

## Технологии

- **Python 3.8+**
- **PyQt6** - графический интерфейс
- **SQLite** - база данных

## Функциональность

### Управление студентами
- Создание, редактирование, удаление студентов
- Поля: фамилия, имя, отчество (опционально), пол, телефон, email (опционально), группа
- Защита от удаления студентов, которые проживали в общежитии

### Управление комендантами
- Создание, редактирование, удаление комендантов
- Поля: фамилия, имя, отчество (опционально), телефон
- Защита от удаления комендантов, участвовавших в заселении/выселении

### Управление корпусами
- Создание, редактирование, удаление корпусов
- Поля: номер корпуса, адрес, количество этажей
- Фильтрация по адресу
- Защита от удаления корпусов с комнатами

### Управление комнатами
- Создание, редактирование, удаление комнат
- Поля: корпус, этаж, номер комнаты, вместимость, площадь (опционально)
- Защита от удаления комнат, в которых проживали студенты

### Заселение
- Заселение студентов с проверками:
  - Вместимость комнаты
  - Соответствие пола заселяемого студента полу уже заселенных
- Автоматическая валидация данных

### Выселение
- Выселение студентов из комнат
- Запись данных о коменданте, оформляющем выселение

## Тестирование

### Тесты методом черного ящика
```bash
python -m pytest tests/test_black_box.py -v
```

### Тесты методом белого ящика
```bash
python -m pytest tests/test_white_box.py -v
```

### Запуск всех тестов
```bash
python -m unittest discover tests
```

## Бизнес-правила

1. Нельзя удалить студента, который проживал в общежитии
2. Нельзя удалить коменданта, участвовавшего в заселении/выселении
3. Нельзя удалить корпус, с которым связаны комнаты
4. Нельзя удалить комнату, в которой проживали студенты
5. Нельзя заселить студента в заполненную комнату
6. Нельзя заселить студента, если его пол не соответствует полу уже заселенных студентов

## Разработка

1. Создайте модель в `app/models.py`
2. Создайте UI в `app/ui/`
3. Добавьте валидацию в `app/utils/validators.py`
4. Напишите тесты в `tests/`
